# 0001: セルフホストの安全レール

- 状態: 採用(2026-07-05)
- 関連: issue #108・#87(見送り時の判断)

## コンテキスト

このリポジトリは atelier プラグインの開発対象であると同時に、atelier 自身で管理される(セルフホスト)。改修対象の工場を自分で運転するため、「壊れた変更が運転中の工場に即波及する」事故を構造的に防ぐ必要がある。前提: 運転はリリース版(バイナリ = Releases のタグ、プラグイン = マーケットプレイス配布の版)で行われ、working tree のコードでは行われない(ブートストラップ分離)。

## 決定

1. **運転は常にリリース版で行う**。working tree でビルドしたバイナリを `.agents/bin/` に置かない。改修の反映は必ずタグ push(バイナリ)/ plugin.json の版上げ + プラグイン更新(hooks・スキル)を経由する
2. **`merge:agent` は当面使わない**。工場自身の変更はすべて人間レビュー・人間マージ(全 PR 人間レーン)
3. **hook / gate / bin に触れる PR は実測テストを必須とする**(ユニットテストに加え、ラッパー経由の実入力での green / red 確認を PR のテスト節に記載する)

## トレードオフ

- 採用しなかった案: working tree ビルドでの運転(反映が速いが、壊れた gate が自分の修正手段まで塞ぐ — #103 の実例)
- 修正の反映がリリースまで遅延するのは意図的(安全側)。緊急時の復旧経路はユーザーの `!` 直接実行(hook を通らない)が常に残る
