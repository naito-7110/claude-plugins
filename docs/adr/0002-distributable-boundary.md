# 0002: 配布物の責任境界

- 状態: 承認(2026-07-05)
- 関連: issue #132・#122(外部状態の撤去)・#129(引き金の撤去)・プリセット merge-policy(補強)
- 還流: プリセット候補として還流済み(#133 → プリセット distributable-boundary)。本 ADR は atelier 固有の適用(`.atelier/`・`docs/`・`.agents/` の列挙、`atelier release` の撤去経緯)を保持する

## コンテキスト

atelier はプラグインとして配布される。2026-07-05、この器のままフル機能(無人自律運転・自動デプロイ)を積んだ結果、2 つの事故に至った。crontab による常駐運転が実測で破綻し(cron の最小 PATH で claude / gh を解決できず、macOS の TCC が crontab 書き込みを拒否した)、かつ配布バイナリに `atelier release`(タグ作成 + push の便利コマンド)という不可逆操作の引き金を含めていた。両者を 5 whys で根本原因まで辿ると、単一の認識の欠落に行き着く。

- **現象**: crontab 運転が破綻し、デプロイの引き金が AI から押せる形で配布されていた
- **なぜ 1**: プラグインがリポジトリの外(マシンの crontab)に状態を書き、配布物に不可逆操作の便利な引き金を含めていたから
- **なぜ 2**: 24 時間の無人自律運転を成立させるには、マシンに常駐機構を仕込み、デプロイまで自動化する必要があると考えたから
- **なぜ 3**: atelier をプラグインという器で作る初期選択があり、その器を固定したまま、要求されるフル機能を積み増そうとしたから
- **なぜ 4**: プラグインは「セッションの拡張」であり、(a) ライフサイクルがセッションに紐づく(アンインストールのフックすら存在しない)、(b) 認証を人間と共有する、(c) 全設置先で同じ挙動が走る。この器では「マシンに永続する状態」も「実行基盤が人間と区別できない引き金」も安全には扱えないから
- **なぜ 5(根本原因)**: 成果物のアイデンティティ(プラグイン / CLI / サービス)ごとに、持ってよい副作用と配ってよい能力の範囲が異なる、という認識を持たず、機能要求から器を逆算せず、器を固定したまま機能を足したから

したがって本 ADR は、機能を積む前に器の境界を定める。範囲を超える機能は、器を変える(別のアイデンティティへ移す)ことで扱う。

## 決定

### 1. 外部状態を持たない

- プラグインはリポジトリ内にしか状態を書かない。書いてよいのは、運用ファイル(`.atelier/`)・一次文書(`docs/`)・作業状態(gitignore 下の `.agents/`)に限る
- リポジトリの外にあるマシンの状態 — OS のスケジューラ(crontab・launchd・systemd タイマー等)、グローバル設定、ホームディレクトリの共有ファイル — を変更しない
- マシン状態の設定が要る機能(定期起動など)は、**人間が設置するためのテンプレートの提供と、読み取り専用の診断**までに留める。設置そのものは人間の操作とする
- 例外: プラグイン機構自身が管理する登録(セッションフックの読み込み等)は、プラグインのコードが書くものではなく、本決定の対象外

### 2. 共有認証下で危険な引き金を配らない

- 実行主体(AI)が人間と同一の認証を共有し、サーバー側が両者を区別できない前提では、不可逆操作(デプロイ・タグ push・公開・削除)の「磨かれた便利コマンド」を配布物に含めない。引き金は人間が生のツール(git 等)で直接引く
- 危険操作を**止める防御**(ゲート)は配布してよい。配らないのは引き金であって、防御ではない
- 判断基準: そのコマンドは「人間の手数を減らす便利さ」を、AI が同じ認証で押せるリスクと引き換えにしていないか。していれば配布物から外し、手順書に置き換える

### 適用の指針

- 新機能を設計するとき、まず「この器(プラグイン)の範囲に収まるか」を問う。収まらないなら機能を捨てるか、器を変える(CLI ツール等の別アイデンティティへ移す)。器を固定したまま範囲外の機能を積まない

## トレードオフ

- **得るもの**: 器の安全性 — 壊れた設定が利用者のマシンや復旧手段を巻き込まない。責務の明確化 — プラグインを消せば痕跡が(リポジトリ内の committed 物を除き)残らない。範囲外の機能は別アイデンティティへ切り出す判断が最初から働く
- **諦めるもの**: 無人自律運転・自動デプロイのような、器の外に踏み出す機能はこのプラグインでは提供できない。定期起動は人間の一手間(テンプレートの設置)を要する
- **緩和策**: 範囲外の機能は器ごと別プロダクトへ移す(この判断が本 ADR の主眼)。診断とテンプレート提供で人間の設置作業を最小化する
