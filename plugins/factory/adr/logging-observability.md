# logging-observability: ログ・可観測性方針

## 適用除外

ログ出力を持たない純ライブラリでは計装の項は該当なし。「秘匿情報を出力しない」はログ・エラーメッセージを 1 行でも出すすべてのコードに適用する。

## コンテキスト

ログの書き方・相関・計測の方針は、後から統一するのが最も難しい横断関心事。エージェントが大量のコードを書く前提では、最初から規範を固定しておく必要がある。特定ベンダーの APM に依存せず、web 標準・業界標準を骨格にする。

## 決定

### ログの書き方

- **構造化ログのみ**を使う。標準出力への生の print・console 出力を本番コードで禁止する
- メッセージテンプレート(構造化パラメータ)を使い、文字列補間でログを組み立てない
- ログは英語で統一する(ユーザー向け文言ではない。i18n-copy の対象外)

### 二重ログの禁止

- 例外・エラーのログは境界の一元ハンドラに集約する。各層で catch してログだけ吐いて再 throw する「二重ログ」を禁止する

### レベル基準(固定)

| レベル | 基準 |
| --- | --- |
| Info | ユースケース完了などの業務イベント(1 リクエストにつき少数) |
| Warn | 予期した失敗・自動回復した異常(error-handling の「予期される失敗」は原則ここ) |
| Error | 対応が必要な予期しない失敗(Unexpected に落ちたもの) |
| Debug 以下 | 開発時のみ。本番の既定レベルは Info |

### 秘匿情報

- 認証情報・トークン・個人情報をログに出力しない。疑わしい値はマスクする(product-security と接続)

### 計装と相関

- 計装の骨格は業界標準(トレース相関は W3C Trace Context、計装は OpenTelemetry 系)とし、エクスポート先はプロジェクトごとに選ぶ。テレメトリ送信は**オプトイン**(未設定でも開発は成立する)
- span はプロセス境界とユースケース境界にだけ張る。細かい関数単位には張らない
- メトリクスはまず標準のもの(レイテンシ・エラー率・外部呼び出し)だけを有効にし、カスタムは必要が生じた issue で追加する

## トレードオフ

- **得るもの**: ベンダー非依存で APM の選定・変更がエクスポーター差し替えに閉じる。レベル基準と二重ログ禁止でログ量とノイズが構造的に抑えられる。相関が標準ヘッダーで自動につながる
- **諦めるもの**: 初期配線のぶん構成要素が増える。送信先未設定の既定ではテレメトリはローカルで消える(開発には支障なし)
